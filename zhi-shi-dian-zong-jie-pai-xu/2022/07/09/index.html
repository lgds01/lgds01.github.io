知识点总结——排序

插入排序

直接插入排序

基本思想：每次将一个待排序的记录按其关键字大小插入前面已排好序的子序列中，直至全部记录排序完成

性能分析：

  • 空间效率：$O(1)$
  • 时间效率：平均$O(n^2)$ 最好$O(n)$ 最差$O(n^2)$
  • 稳定性：稳定
  • 适用性：适用于顺序存储和链式存储
def insert_sort(arr):
    for i in range(1, len(arr)):
        insert_num = arr[i]
        j = i - 1
        while j >= 0 and arr[j] > insert_num:
            arr[j + 1] = arr[j]
            j -= 1
        arr[j + 1] = insert_num

折半插入排序

基本思想：先用折半查找得到插入位置，然后再统一后移元素

特点：

  • 在直接插入排序的基础上减少了比较的次数，约为$O(nlogn)$

性能分析：

  • 空间效率：$O(1)$
  • 时间效率：平均$O(n^2)$ 最好$O(n)$ 最差$O(n^2)$
  • 稳定性：稳定
  • 适用性：适用于顺序存储
def binary_insert_sort(arr):
    for i in range(1, len(arr)):
        insert_num, l, r = arr[i], 0, i - 1
        while l <= r:
            m =int( (l + r) / 2)
            if arr[m] > arr[i]:
                r = m - 1
            else:
                l = m + 1
        for j in range(i, l, -1):
            arr[j] = arr[j - 1]
        arr[l] = insert_num

希尔(shell)排序

基本思想：先将待排序表中相隔某个增量的记录组成一个子表，对各个子表分别进行直接插入排序，当整个子表基本有序时，对整个表进行直接插入排序

特点：

  • 基于直接插入排序在待排序列接近正序时时间复杂度为$O(n)$的特点改进而来
  • 步长逐渐减小，从而使得待排序列逐渐有序

性能分析：

  • 空间效率：$O(1)$
  • 时间效率：分析困难
  • 稳定性：不稳定（当相同关键字的记录被划分到不同子表时）
  • 适用性：适用于顺序存储
def shell_sort(arr):
    d = int(len(arr) / 2)
    while d >= 1:
        for i in range(d, len(arr), 1):
            if arr[i] < arr[i - d]:
                insert_num, j = arr[i], i - d
                while j >= 0 and arr[j] > insert_num:
                    arr[j + d] = arr[j]
                    j -= d
                arr[j + d] = insert_num
        d = int(d / 2)

交换排序

冒泡排序

基本思想：从后往前两两比较相邻元素的值，若为逆序，则交换，直至存在一趟冒泡没有发生交换

特点：

  • 每趟冒泡，都有至少有一个元素在其最终位置

性能分析：

  • 空间效率：$O(1)$
  • 时间效率：平均$O(n^2)$ 最好$O(n)$ 最差$O(n^2)$
  • 稳定性：稳定
  • 适用性：顺序存储
def bubble_sort(arr):
    for i in range(0, len(arr) - 1):
        flag = True
        for j in range(len(arr) - 1, i, -1):
            if arr[j] < arr[j - 1]:
                arr[j - 1], arr[j] = arr[j], arr[j - 1]
                flag = False
        if flag:
            break

快速排序

基本思想：基于分治法，选定一个枢轴，将待排序列分为小于枢轴和大于枢轴的两个部分，然后继续在各个部分进行上述操作，直至每个部分只有一个记录

特点：

  • 每进行一趟排序，都至少有一个元素在其最终位置上
  • 内部排序算法中平均性能最优的算法
  • 当枢轴将待排序列划分成两个平衡的部分时，快速排序达到最好性能（包括空间和时间），而当枢轴将待排序列划分成 n-1 和 0 个元素的两部分时，快速排序达到最差性能

性能分析：

  • 空间效率：平均$O(logn)$ 最好$O(logn)$ 最差$O(n)$
  • 时间效率：平均$O(nlogn)$ 最好$O(nlogn)$ 最差$O(n^2)$
def quick_sort(arr, left, right):
    if left >= right:
        return
    l, r = left, right
    while (l < r):
        while l < r and arr[r] >= arr[left]:
            r -= 1
        while l < r and arr[l] <= arr[left]:
            l += 1
        arr[l], arr[r] = arr[r], arr[l]
    arr[r], arr[left] = arr[left], arr[r]
    quick_sort(arr, left, r - 1)
    quick_sort(arr, r + 1, right)

选择排序

简单选择排序

基本思想：每一趟在后面 n-i+1 个待排序元素中选取关键字最小的元素，然后与第 i 个元素交换，直至第 n-1 趟整个排序表有序

特点：

  • 每进行一趟排序，都有一个元素在其最终位置上
  • 比较次数与序列的初始状态无关，时钟都是$n(n-1)/2$

性能分析：

  • 空间效率：$O(1)$
  • 时间效率：$O(n^2)$
  • 稳定性：不稳定（交换时，可能会导致相同关键字的元素之间相对位置发生改变）
def simple_select_sort(arr):
    for i in range(0, len(arr) - 1):
        index, min = i, arr[i]
        for j in range(i + 1, len(arr)):
            if arr[j] < min:
                index, min = j, arr[j]
        arr[i], arr[index] = min, arr[i]

堆排序

基本思想：利用堆的性质，不断从堆中删除堆顶元素，从而形成有序序列

特点：

  • 每进行一趟排序，都有一个元素在其最终位置上

性能分析：

  • 空间效率：$O(1)$
  • 时间效率：$O(nlogn)$
  • 稳定性：不稳定
def adjust_heap(arr, k, length):
    i = 2 * k + 1
    while i < length:
        if i + 1 < length and arr[i] < arr[i + 1]:
            i+=1
        if arr[k] >= arr[i]:
            break
        arr[k], arr[i] = arr[i], arr[k]
        k, i = i, 2 * i + 1


def create_heap(arr):
    for i in range(int(len(arr) / 2 - 1), -1, -1):
        adjust_heap(arr, i, len(arr))


def heap_sort(arr):
    create_heap(arr)
    for i in range(len(arr)-1, 0, -1):
        arr[i], arr[0] = arr[0], arr[i]
        adjust_heap(arr, 0, i)

归并排序（分治排序）

特点：

  • N 个元素进行 k 路归并排序时，排序的趟数 m 满足$k^m=N$，从而$m=ceil(log_kN)$

性能分析：

  • 空间效率：$O(n)$
  • 时间效率：$O(nlogn)$
  • 稳定性：稳定
def merge(arr, left, mid, right):
    tmp = [item for item in arr]
    l, m = left, mid + 1
    while l <= mid and m <= right:
        if tmp[l] <= tmp[m]:
            arr[left], left, l = tmp[l], left + 1, l + 1
        else:
            arr[left], left, m = tmp[m], left + 1, m + 1
    while l <= mid:
        arr[left], left, l = tmp[l], left + 1, l + 1
    while m <= right:
        arr[left], left, m = tmp[m], left + 1, m + 1


def merge_sort(arr, left, right):
    if left >= right:
        return
    mid = int((left + right) / 2)
    merge_sort(arr, left, mid)
    merge_sort(arr, mid + 1, right)
    merge(arr, left, mid, right)

基数排序

特点：

  • 不是基于比较的排序
  • 适用于关键字由 d 元组组成的序列

性能分析：

  • 空间效率：$O(n+r)$ r 为队列个数
  • 时间效率：$O(d(n+r))$ d 为分配和收集趟数
  • 稳定性：稳定
